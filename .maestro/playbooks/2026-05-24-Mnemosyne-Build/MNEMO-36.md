Phase 36 (see `MNEMO-00-ROADMAP.md`). Goal: the per-agent home — a tabbed shell hosting **Chat**,
**Reports**, **Audit**, **Settings**, and **Metadata**, with the conversation UI mounted under the Chat
tab. This phase builds the shell + the Settings/Metadata tabs in full; Reports and Audit get placeholder
tab panels that later phases (MNEMO-41 report viewer, MNEMO-37 glass cockpit) fill in. Depends on
MNEMO-34 (agent list/create + `frontend/src/api/agents.ts`) and MNEMO-35 (conversation UI components:
`ConversationList`, `ConversationPage`). Per `docs/PRD.md` v0.5 §6.6 (agent detail: chat history, reports,
audit log, settings, creation date, metadata; "brain size" metric = neurons + synapses) and §5
(Discovery → Build → Operation; settings cover schedule + template).

Conventions: the detail shell lives in `frontend/src/pages/agents/AgentDetailPage.tsx` with tab panels in
`frontend/src/components/agents/tabs/`. Tabs are routed (URL-addressable, e.g.
`/agents/:agentId/chat`, `/settings`, `/metadata`, `/reports`, `/audit`) so deep-linking and back-button
work. Settings edits go through `updateAgent` (MNEMO-05 `PATCH /agents/:id`). Separate component code /
tests / test-runs.

- [x] Create a reusable tabs primitive `frontend/src/components/ui/Tabs.tsx` (+ `.module.css`) and export it from the UI barrel: a router-aware tab strip (each tab is a `NavLink` to a sub-route) with an active underline/indicator, keyboard-navigable (arrow keys + roving tabindex), ARIA `role="tablist"`/`tab`/`tabpanel`. It renders the tab strip and lets the routed `<Outlet/>` render the panel. Mobile: the strip is horizontally scrollable. Keep it generic (label + path + optional badge count per tab).
  - **Deviation (intentional):** `Tabs.tsx` already existed in the UI library as a generic **content-based** tabs primitive (in-place panels, no router), used by the dev gallery and covered by `Tabs.test.tsx`. Rather than overload/break it, I added the router-aware variant as a **separate primitive `frontend/src/components/ui/RoutedTabs.tsx`** (+ `.module.css`), exported from the barrel as `RoutedTabs`/`RoutedTabSpec`/`RoutedTabsProps`. Each tab is a `<Link>` whose active state is derived from the URL via `useResolvedPath` + `matchPath` (prefix match, so Chat stays active for `/chat/:id`), giving the active underline indicator, `aria-current`/`aria-selected`, roving `tabindex`, and arrow/Home/End keyboard nav (selection follows focus → navigates). It renders the strip + a routed `<Outlet/>` (with an optional `outletContext` pass-through), the strip is horizontally scrollable on mobile, and it takes a generic `{ label, to, badge? }` per tab. Catalog entry added to the dev gallery (wrapped in a self-contained `MemoryRouter`). Tested in `components/ui/__tests__/RoutedTabs.test.tsx` (the spec's `Tabs.test.tsx` name is taken by the existing content primitive).
