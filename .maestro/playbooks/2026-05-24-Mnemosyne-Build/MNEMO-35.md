Phase 35 (see `MNEMO-00-ROADMAP.md`). Goal: the core interactive surface — threaded conversations with an
agent, assistant responses **streamed** from the per-agent DO, plus conversation rename and search.
Depends on MNEMO-15 (agent loop: `streamText`/`generateText` in the `AIChatAgent` DO with message-history
persistence, exposing a streaming chat endpoint per agent) and MNEMO-32 (scaffold). Per `docs/PRD.md` v0.5
§6.5 (threaded conversations with search + rename; conversation-centric navigation; agent avatars;
mobile-responsive parity; filter by agent / time / keyword) and §7.1 (the harness is the Vercel AI SDK
`streamText` loop hosted by `AIChatAgent`, streamed to the client).

Conventions: the streaming uses the Vercel AI SDK UI-message stream protocol that `AIChatAgent`/
`toUIMessageStreamResponse` emits (SSE-style chunked stream) — consume it with the `ai`/`@ai-sdk/react`
client rather than hand-parsing, unless the backend's contract requires a thin custom reader. Conversation
pages live in `frontend/src/pages/conversations/`, chat components in `frontend/src/components/chat/`,
conversation API calls in `frontend/src/api/conversations.ts`. A "conversation" is one thread within one
agent. Separate component code / tests / test-runs.
