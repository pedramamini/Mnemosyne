Phase 42, Track G — Frontend (see `MNEMO-00-ROADMAP.md`). Goal: a management/metrics dashboard to
**list, filter, and search agents**, each card showing key metadata and the **brain-size metric** (neurons
+ synapses). Depends on **MNEMO-05** (agent registry CRUD API: list/get agents), **MNEMO-09** (graph +
brain-size metric endpoint), **MNEMO-34** (agent list + create wizard — this phase upgrades the bare list
into a metrics dashboard and reuses the create wizard), and the scaffold (MNEMO-32). Per `docs/PRD.md` v0.5
§6.6: list/filter/search agents; agent detail metadata; "brain size" metric = neurons (files) + synapses
(parsed links), backed by the real graph.

Conventions: reuse the agents API client and the create-wizard entry point from MNEMO-34 — do not
re-implement agent CRUD. Add a brain-size client over MNEMO-09 (or reuse `frontend/src/api/graph.ts` from
MNEMO-40 if that already exposes brain-size). The dashboard is a top-level page under
`frontend/src/pages/`; cards/filters live under `frontend/src/components/dashboard/`. Brain-size is fetched
per agent lazily/in-parallel and degrades gracefully (show "—" while loading or on error — never block the
whole list on one agent's metric). Filtering/search happens client-side over the listed agents unless
MNEMO-05 already provides query params (prefer server-side filter if available). Tests use vitest +
@testing-library/react with the API client mocked.
