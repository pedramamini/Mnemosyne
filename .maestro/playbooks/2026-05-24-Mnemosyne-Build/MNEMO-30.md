# MNEMO-30 — Lifecycle: Build / provisioning flow (spec → live agent)

Phase 30 (see `MNEMO-00-ROADMAP.md`). Goal: the **Build** stage of the lifecycle — turn a finalized
Discovery spec (MNEMO-29) into a **live, operable agent**. Build provisions the sandbox filesystem,
assembles the system prompt, enables the tool set, sets schedule defaults, and applies the entity
template, then flips the agent to `active`/`operational` so it can run research turns. Depends on
MNEMO-05 (agent registry CRUD + DO/D1 wiring), MNEMO-06 (sandbox provisioning: `exec`/`readFile`/
`writeFile`, warm/idle lifecycle, R2 persistence) and MNEMO-29 (the Discovery spec). It also touches
MNEMO-07's brain layout/`git init` and MNEMO-13's `getModel()` indirectly. Per `docs/PRD.md` v0.5 §5(2):
"Platform provisions the agent: filesystem, system prompt, enabled tools (web search, tool execution),
schedule defaults. Agent is configured with its template (vendor/product/investor/founder)."

Conventions: Build is **orchestrated from the DO** (PRD §7.1 topology A) — the DO is the always-home;
it spins the sandbox up on demand to lay down the initial filesystem, then lets it idle (PRD §8.4). Build
is **idempotent and resumable**: it records a `buildStatus` with per-step flags in `agent_meta` so a
re-run skips completed steps (provisioning can fail mid-way — a half-built sandbox must be safe to retry).
The template fragments themselves land in MNEMO-31; this phase defines the **template application
interface** and ships a minimal default so Build is testable end-to-end now, with MNEMO-31 filling in the
four real templates behind the same interface. Brain filesystem paths follow MNEMO-07's layout
(`/brain`, `/brain/tools/`, `/brain/reports/`, `/brain/notes/`). No raw model loop here — Build is
provisioning, not research.
