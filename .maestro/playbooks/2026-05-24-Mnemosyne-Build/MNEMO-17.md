# MNEMO-17 — Harness & Tools: Web research tools (search + fetch)

Phase 17 (see `MNEMO-00-ROADMAP.md`). Goal: the web-research tools — `webSearch` and `webFetch` — added
to the tool registry, carrying Crema's safety rails: a `BLOCKED_HOSTS` set hard-blocked at the fetch
layer, a 15s timeout, and a 200KB content cap. Depends on MNEMO-16 (tool framework: `ToolContext`,
`buildTools`, `spillIfLarge`, audit emit). Per `docs/PRD.md` v0.5 §6.3 (web search + fetch with
time/size safety caps — carry Crema's `BLOCKED_HOSTS`, 15s timeout, 200KB cap) and the safety-rails
catalog in `docs/crema-architecture-reference.md` §12.

Conventions: fetched/searched output obeys the same large-output-to-FS-path discipline as MNEMO-16 — a
fetched page over the inline threshold is spilled to `/brain/.tool-out/` and the tool returns a PATH +
preview, not the blob (per `docs/PRD.md` §7.1). `BLOCKED_HOSTS` is enforced on the resolved host of
every URL (and on redirect targets), 15s via `AbortController`, 200KB via a streamed byte counter that
aborts past the cap. Each web action emits a `source.read` audit event so the glass cockpit shows "read
N sources." HTML is reduced to readable text before spilling (keep raw HTML out of the loop context).
