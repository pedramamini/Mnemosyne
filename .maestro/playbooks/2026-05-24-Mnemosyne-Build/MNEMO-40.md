Phase 40, Track G — Frontend (see `MNEMO-00-ROADMAP.md`). Goal: an interactive **brain graph map** that
visualizes the agent's memory as **neurons** (files) and **synapses** (`[[wikilink]]` edges) and **grows
over time** as the brain does. Depends on **MNEMO-09** (graph retrieval tool / endpoint: traversal queries
+ search + brain-size metric — this exposes nodes/edges), **MNEMO-36** (agent detail page tabs — adds a
"Graph" tab), and the scaffold (MNEMO-32). Per `docs/PRD.md` v0.5 §6.2: parse `[[wikilinks]]` into a real
graph and expose a **visible map that grows**; §4: files = neurons, parsed links = synapses, "brain size" =
neuron count + synapse count.

Conventions: render the graph with a force-directed layout. Use a lightweight, well-maintained graph
library compatible with React + Vite (e.g. `react-force-graph-2d` / a canvas force-graph, or d3-force +
canvas) — pick one and pin it; prefer canvas over SVG so large brains stay performant. Graph data comes
from the MNEMO-09 endpoint via a new API client module. The visualization is presentational and reads
nodes/edges from props; data fetching lives in a hook. Clicking a neuron deep-links into the brain explorer
file (MNEMO-38). Tests use vitest + @testing-library/react and assert data transforms and wiring, NOT pixel
output (mock/stub the graph renderer so tests don't depend on canvas).
