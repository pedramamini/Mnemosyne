# MNEMO-13 — Harness & Tools: LLM provider resolver `getModel()`

Phase 13, opening Track C — Harness & Tools (see `MNEMO-00-ROADMAP.md`). Goal: a single
model-agnostic resolver, `getModel()`, that returns a Vercel-AI-compatible `LanguageModel` resolved
**per-user** from their profile — Workers AI Qwen3-30B as the free default, with per-user BYOK
(`{provider, model, key}`) for OpenRouter / Anthropic / OpenAI. Depends on MNEMO-02 (D1 `accounts`
table + `src/db`) and MNEMO-05 (agent registry, which carries the owning `account_id`). Per
`docs/PRD.md` v0.5 §7.2 + §8.5: free default `@cf/qwen/qwen3-30b-a3b-fp8` (confirmed function-calling
+ reasoning); BYOK saved in the user's profile; OpenRouter primary, Anthropic/OpenAI direct as asked.
This phase resolves the model only — AI Gateway routing + spend caps + secret custody come in MNEMO-14.

Conventions: resolver lives in `src/llm/`. It speaks only the `ai` SDK's `LanguageModel` interface so
swapping providers/models is config, not code (mirrors Crema's `llm.ts`, but Crema's is global-env and
ours is per-user — see `docs/crema-architecture-reference.md` §4). BYOK keys are persisted but NOT
echoed back to clients. Provider/model values are validated with Zod; an unknown or unconfigured
provider falls back to the free Workers AI default rather than throwing.
