# MNEMO-21 — Audit Log: emit events across the agent loop and tools

Phase 21 (see `MNEMO-00-ROADMAP.md`), Track D. Goal: make the agent actually *write to* the glass
cockpit — instrument the agentic loop, the tool layer, the memory layer, and the reporting path to emit
the higher-level **productivity events** (`session.*`, `source.read`, `memory.wrote/linked/consolidated`,
`tool.authored/ran`, `report.generated`, `chart.rendered`, `narration`, `error`). Depends on MNEMO-15
(the `streamText`/`generateText` loop in `MnemosyneAgent`), MNEMO-16 (tool framework), and MNEMO-20 (the
`AuditLog` DO + `emit` RPC). Per `docs/PRD.md` v0.5 §6.7 and §7.1: events are a *productivity stream*, not
a raw token/tool dump; `level` is the altitude control (`milestone` for the calm default stream, `info` for
"show the work", `error` for failures); reasoning parts feed the "show the work" view.

Conventions: never emit raw model tokens or full tool blobs into the audit log — emit a human summary plus
a small structured `payload` (the spike already JSON-stores `payload`). All emission goes through one
typed `AuditEmitter` so call sites never touch the DO stub directly and so emit failures can never break
the loop (audit is observability, not control flow). Choose `level` deliberately per event type per the
rubric in the first task. `sessionId` groups every event from one research run.
