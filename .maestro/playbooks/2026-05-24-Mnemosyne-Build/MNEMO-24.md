# MNEMO-24 — Reporting: report generation (markdown + Obsidian front matter + embedded PNG)

Phase 24 (see `MNEMO-00-ROADMAP.md`), Track E. Goal: assemble a finished **report artifact** — Obsidian-style
YAML front matter + markdown body + **embedded PNG charts** (from MNEMO-23) — produced on demand or on
schedule, written to the brain FS, and audited. Depends on MNEMO-23 (`src/reports/` Code Interpreter +
chart→PNG pipeline) and MNEMO-10 (memory write/append + consolidation, the source of the findings the report
summarizes). Per `docs/PRD.md` v0.5 §6.4: markdown output with Obsidian front matter, computed charts
rendered to PNG embedded so one artifact works across web/email/SMS. (Archive to R2 + D1 metadata is MNEMO-25;
delta/diff is MNEMO-26 — this phase builds the artifact itself.)

Conventions: a report is a self-contained markdown file plus its referenced PNG assets, both in the brain FS
(so they live in git, §6.9) before MNEMO-25 lifts them to R2. Front matter is a typed object serialized to YAML
deterministically (stable key order) so MNEMO-26 can diff two reports. Charts are referenced by relative path
in the markdown **and** their bytes are carried alongside so MNEMO-25 can store/inline them for email/SMS. The
report generator is pure assembly over inputs (findings + chart specs) — it does not itself decide *what* to
research; that comes from the loop / schedule.

Keep YAML emission minimal/dependency-light (a small deterministic serializer or a pinned YAML lib already in the tree); document the stable-order requirement with a `// MNEMO-26` comment.
  - **Done.** `ReportFrontMatter` reuses the registry `AgentTemplate` enum for `template` (lockstep, no drift). `serializeFrontMatter` parses the input (applies `type`/`tags` defaults) then emits keys in a fixed `KEY_ORDER` (documented `// MNEMO-26`), omitting absent optionals → byte-identical output for the same input. Small dependency-light deterministic emitter (no YAML lib in the tree): `tags` renders as a block list (or `[]`); scalars are quoted only when ambiguous (colons/dates/bool-/number-like), with `\`/`"`/newline escaping.

  - **Done.** `buildReportMarkdown(input, deps)` composes `serializeFrontMatter(fm)` → `# title` → per-section `## heading`+body → optional embedded chart → a `## Related` block of `[[wikilink]]`s from `input.related`.
