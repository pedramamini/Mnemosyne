Phase 41, Track G — Frontend (see `MNEMO-00-ROADMAP.md`). Goal: a **report viewer** that renders an
agent's reports — markdown + Obsidian front matter + embedded PNG charts — plus a **full-text search** UI
across reports. Depends on **MNEMO-25** (report archive + storage backend: R2 blobs, D1 metadata,
retrieval, and report list/search), **MNEMO-36** (agent detail page tabs — adds/owns a "Reports" tab), and
the scaffold (MNEMO-32). Per `docs/PRD.md` v0.5 §6.4: markdown output with Obsidian front matter; computed
charts rendered to **PNG** (chosen so one artifact embeds across web/email/SMS); web-based report viewing +
full-text search.

Conventions: render markdown with a vetted React markdown renderer (e.g. `react-markdown` + `remark-gfm`),
sanitizing HTML. Front matter is parsed separately and shown as a metadata header (do NOT render raw YAML in
the body). Embedded PNG charts are referenced by the report markdown / its asset list and resolved to R2
blob URLs from the MNEMO-25 retrieval endpoint. Report list + search hit the MNEMO-25 API. Components live
under `frontend/src/components/reports/`; the page is the Reports tab under `frontend/src/pages/agent/`.
Tests use vitest + @testing-library/react with the API client mocked.
