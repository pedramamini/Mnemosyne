# MNEMO-04 — Foundation: Per-agent Durable Object skeleton

Phase 4 (see `MNEMO-00-ROADMAP.md`). Goal: the always-cheap per-agent home — a `MnemosyneAgent`
Durable Object extending `AIChatAgent` that survives hibernation and holds the agent's settings,
schedule, and DO-SQLite state. This is the "always-home" half of the DO-warm / sandbox-ephemeral
split in `docs/PRD.md` v0.5 §7.4 and §8.4 (the DO holds chat history, settings, schedule, and the
memory/audit index so search and brain-size work *without* waking the container). No agentic loop,
no tools, no sandbox yet — those land in MNEMO-06 and Track C. Depends on MNEMO-01 (Worker scaffold,
`wrangler.toml` binding-stub pattern, `src/env.ts`, vitest workers pool).

Conventions: one DO instance per agent via `env.AGENT.idFromName(agentId)` — no allocation logic
(mirrors Crema, see `docs/crema-architecture-reference.md` §1). The DO is declared as a
`new_sqlite_classes` migration so `ctx.storage.sql` is available. All DO-SQLite access goes through a
thin typed module so later phases (memory index in MNEMO-08, schedule in MNEMO-27) extend one schema
init path rather than scattering DDL. Settings/schedule are persisted to DO storage so they rehydrate
after hibernation.
