# MNEMO-08 — Sandbox & Brain: `[[wikilink]]` parser & neuron/synapse graph index

Phase 8 (see `MNEMO-00-ROADMAP.md`). Goal: turn the brain's files into a *real graph* — parse
`[[wikilink]]`-style links out of note content and maintain a neuron/synapse index in DO SQLite
(files = neurons, parsed links = synapses) so search, traversal, and brain-size work *without* waking
the container. This is the heart of the product thesis. Depends on MNEMO-06 (read files from the
sandbox) and MNEMO-07 (brain layout — where notes live). Per `docs/PRD.md` v0.5 §4 ("Files on the
agent's filesystem = neurons. `[[wikilink]]`-style links parsed into a real graph = synapses... brain
size = neuron count + synapse count") and §6.2 ("parse `[[wikilinks]]` into a real graph; expose a
visible map that grows; provide graph traversal as a retrieval tool"). Architecture: the memory
*index* lives in DO SQLite (PRD §7.4 — "memory index ... so search/brain-size work without waking the
container").

Conventions: the index is metadata only — neuron rows reference a `path` on the sandbox FS, they do
not store file *content* (content lives in the brain; the DO holds the graph). The parser is pure and
unit-tested independently of any sandbox or DO. The index DDL extends the same `src/agent/sql.ts`
schema-init path established in MNEMO-04 (one schema-init chokepoint per DO). Links resolve by note
title/slug; an unresolved `[[link]]` (target note doesn't exist yet) is still recorded as a synapse to
a *dangling* target so the graph can surface "wanted but unwritten" notes. Re-indexing a neuron is
idempotent (delete-then-insert its outgoing synapses) so repeated writes don't duplicate edges.
