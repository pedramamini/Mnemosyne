Phase 33 (see `MNEMO-00-ROADMAP.md`). Goal: the passwordless magic-link login experience on the
frontend — a request-link screen, the callback landing, app-wide session state, and a protected-route
wrapper that gates the rest of the app. Depends on MNEMO-03 (backend magic-link routes: `POST /auth/request`,
`GET /auth/callback?token=`, `POST /auth/logout`, cookie session + `requireAuth`) and MNEMO-32 (scaffold:
`apiFetch`/`ApiError`/`isUnauthorized`, `AppShell`, UI primitives, router). Per `docs/PRD.md` v0.5 §6.1
(magic-link via Resend, time-limited single-use tokens; one account → many agents) and §8.3 (auth low-risk,
well-trodden pattern).

Conventions: the session is a cookie set by the Worker (HttpOnly), so the SPA never sees the token — it
infers auth state from an authenticated probe endpoint, not from reading a cookie. Auth UI lives in
`frontend/src/pages/auth/`, shared auth logic in `frontend/src/auth/`, and auth API calls in
`frontend/src/api/auth.ts`. The backend's `/auth/request` always returns 200 (anti-enumeration), so the
UI shows a neutral "check your email" state regardless. Separate component code / tests / test-runs.
