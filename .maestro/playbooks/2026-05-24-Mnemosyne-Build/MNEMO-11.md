# MNEMO-11 — Sandbox & Brain: Brain explorer backend

Phase 11 (see `MNEMO-00-ROADMAP.md`). Goal: the backend that makes the brain browsable, editable, and
downloadable from the web — list/read/write/create/delete files anywhere under `/brain`, plus a
whole-brain archive (zip/tar) download endpoint. This is the API the Brain Explorer UI (MNEMO-38)
consumes. Depends on MNEMO-06 (sandbox FS client) and MNEMO-07 (brain layout + git auto-commit so
explorer edits are versioned too). Per `docs/PRD.md` v0.5 §6.9: "the agent's filesystem (its brain) is
browsable and editable from the web UI: view, edit, create, and delete files/notes directly, and
download the entire brain as an archive (zip/tarball)."

Conventions: this is a *general file* explorer over `/brain` (not just `notes/*.md`) — it can touch
`tools/`, `reports/`, anything in the tree — but every path is validated to stay inside `/brain` (no
`..`/absolute escape) via a shared guard. Human edits from the explorer are first-class memory writes:
when an edited/created/deleted path is a note, reindex the graph (MNEMO-08) and always `autoCommit`
(MNEMO-07) so the UI's edits show up in history and graph just like agent writes. All FS access goes
through the MNEMO-06 client wrapper. Reads are bounded (size cap) and binary files are handled (don't
try to return a 50MB PNG as a JSON string — stream or base64 with a size guard).
