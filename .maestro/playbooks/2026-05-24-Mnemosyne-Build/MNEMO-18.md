# MNEMO-18 — Harness & Tools: Terminator tool & final-report schema

Phase 18 (see `MNEMO-00-ROADMAP.md`). Goal: deliberate loop exit — a **terminator tool** whose
`inputSchema` **is** the final-report Zod schema, captured via a closure so the DO reads the structured
result after the loop returns. Depends on MNEMO-15 (the loop + step budgets) and MNEMO-16 (the tool
registry + `ToolContext`). Per `docs/PRD.md` v0.5 §6.3 + §7.1: for deep research the step budget is high
(~50–200) and exit is deliberate via a terminator tool whose input schema is the final-report schema.
Mirrors the "terminator-tool-as-schema" pattern in `docs/crema-architecture-reference.md` §6 — cleaner
than JSON-mode-and-pray; if the model finishes without calling it, that's a detectable soft-fail.

Conventions: the terminator is registered only for headless/deep-research runs (where a structured
final report is the goal), not necessarily every interactive turn. The result is captured into a
per-run `sink` closure (the §6 pattern); after `generateText`/`streamText` returns, the DO reads the
sink. Calling the terminator does not itself stop the loop mid-step — but the model is instructed to
call it exactly once as its final action, and `stopWhen` is extended to also stop when the terminator
has fired (so the loop ends promptly on a deliberate exit). A terminated run emits a
`report.generated` audit event; a run that hits the step ceiling without terminating emits an `error`-
level audit note (the detectable soft-fail).
