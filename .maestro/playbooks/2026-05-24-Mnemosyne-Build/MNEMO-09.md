# MNEMO-09 — Sandbox & Brain: Graph retrieval & brain-size metric

Phase 9 (see `MNEMO-00-ROADMAP.md`). Goal: make the neuron/synapse graph *useful* — traversal
queries, search over neurons, and the "brain size" metric — all served from the DO SQLite index built
in MNEMO-08, so they answer without waking the container. This is the retrieval substrate the agent
reasons over (consumed as a tool in Track C / MNEMO-16) and the metric the UI surfaces (MNEMO-42).
Depends on MNEMO-08 (`GraphIndex`, neurons/synapses tables, the schema-init path). Per
`docs/PRD.md` v0.5 §6.2 ("provide graph traversal as a retrieval tool"), §4 + §6.6 ("brain size =
neuron count + synapse count, backed by the real graph"), and §7.4 (index lives in DO SQLite so
search/brain-size work without waking the sandbox).

Conventions: read-only over the MNEMO-08 tables — this phase adds query methods, not new writes. All
queries are parameterized and bounded (explicit `LIMIT` / max depth) so a pathological graph can't run
away — same discipline as the audit store's `MAX_LIMIT`. Traversal is breadth-first with a depth cap
and visited-set to avoid cycles. Search here is over neuron titles/slugs in the index; full-text over
note *content* stays on the sandbox FS (`grep`) and is a Track-C tool — keep that boundary clear in a
comment. Methods live on the same `GraphIndex` class (extend it) so there's one graph access point.
