# MNEMO-07 — Sandbox & Brain: Brain filesystem layout & git versioning

Phase 7 (see `MNEMO-00-ROADMAP.md`). Goal: lay down the agent's brain on the sandbox filesystem and
turn it into a git repo — a standard directory layout under `/brain`, `git init` on first
provisioning, and an auto-commit helper invoked on every memory write and on each consolidation pass.
This is the safety net the rest of Track B depends on: per-file diff/restore (MNEMO-12) and the
"versioned + diffed before commit" guarantee consolidation requires (MNEMO-10). Depends on MNEMO-06
(sandbox client wrapper + lifecycle). Per `docs/PRD.md` v0.5 §6.9 (the brain is a git repo inside the
sandbox, auto-committed on writes and on each consolidation pass; commit history + per-file diffs +
one-click restore; git-in-sandbox gives clean per-file diff/restore, R2 versioning is the coarser
backstop) and §6.2 (memory = files persisted on the FS; `/brain/tools/` holds self-authored tools).

Conventions: all brain paths live under `/brain`. Notes/neurons are `.md` files; self-authored tools
live in `/brain/tools/`; reports archive under `/brain/reports/`. All git and FS operations go
through the sandbox client wrapper from MNEMO-06 (`run` / `readFile` / `writeFile` / `mkdir`) — never
the raw SDK. Git is configured with a stable, non-personal identity (the agent's own). Auto-commit is
the single chokepoint every write path calls, so history is complete and diffs are meaningful. Keep
commit messages structured (a parseable prefix) so MNEMO-12 can categorize history.
