# MNEMO-25 — Reporting: report archive + storage (R2 blobs, D1 metadata, retrieval API)

Phase 25 (see `MNEMO-00-ROADMAP.md`), Track E. Goal: archive generated reports durably — upload the markdown
+ PNG assets to **R2**, record **D1 `reports` metadata** (using the table from MNEMO-02), and expose an
authenticated **retrieval API** (list / get markdown / get a PNG asset) for the web report viewer (MNEMO-41).
Depends on MNEMO-02 (D1 schema: the `reports` table — `id`, `agent_id`, `title`, `r2_key`, `front_matter`,
`created_at` — plus the `src/db` typed access layer and `createReport`/`listReportsByAgent` helpers) and
MNEMO-24 (`generateReport` returns `{ markdown, frontMatter, brainPath, assets }`). Per `docs/PRD.md` v0.5
§6.4 and §7.4: report metadata in D1, blobs in R2, web-based report viewing + full-text search.

Conventions: R2 is the durable store of record for the published artifact (the brain-FS copy from MNEMO-24
remains the git-versioned working copy). One R2 "prefix" per report (`agents/<agentId>/reports/<reportId>/`)
holding `report.md` + `assets/*.png`, so a report is a self-contained object group. D1 holds only metadata
(no blobs). All retrieval routes are `requireAuth` + ownership-gated (reuse the MNEMO-05 / MNEMO-22
`assertOwnsAgent` guard). The R2 binding is added here if not already present.
