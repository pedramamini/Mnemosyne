# MNEMO-22 — Audit Log: the audit API (SSE backfill, filter, search, altitude)

Phase 22 (see `MNEMO-00-ROADMAP.md`), Track D. Goal: expose the glass cockpit over HTTP — an SSE stream
endpoint with **`sinceSeq` reconnect backfill**, a filterable query endpoint (type / level / session / time),
an **FTS5 search** endpoint, and an explicit **altitude** parameter (`milestone` default, `info` for "show the
work", `error`). Depends on MNEMO-20 (the `AuditLog` DO with `emit`/`query`/`search` RPC + the `SseHub`).
Per `docs/PRD.md` v0.5 §6.7: streamable via SSE with a `sinceSeq` cursor for reconnect backfill, filterable
and searchable, with `level` as the altitude control enabling narrated progressive disclosure.

Conventions: routes mount under `/agents/:agentId/audit/*`, all behind `requireAuth` (MNEMO-03) and an
ownership check (the requesting account must own the agent — reuse the MNEMO-05 agent-registry lookup).
Query params are Zod-validated and passed straight into the store's existing `AuditQuery` shape (do not
re-implement filtering — the spike already does it; the DO just forwards). The DO already uses the event
`seq` as the SSE `id:` (MNEMO-20), so reconnect backfill keys off the standard `Last-Event-ID` header
**and** an explicit `?sinceSeq=` query param.
