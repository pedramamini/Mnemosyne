# MNEMO-12 — Sandbox & Brain: Brain versioning backend

Phase 12 (see `MNEMO-00-ROADMAP.md`). Goal: expose the brain's git history to the web — commit
history, per-file diffs, and one-click restore to a prior revision — the safety net consolidation
(MNEMO-10) and a bad self-authored-tool write require, and the backend for the Brain Versioning UI
(MNEMO-39). Depends on MNEMO-07 (the `/brain` git repo + `autoCommit` chokepoint + structured commit
messages). Per `docs/PRD.md` v0.5 §6.9: "the UI exposes commit history, per-file diffs, and one-click
restore to a prior revision. This doubles as the safety net consolidation requires (versioned + diffed
before commit) and a guard against a bad self-authored-tool write." Git-in-sandbox gives clean
per-file diff/restore; R2 object versioning (MNEMO-06) is the coarser backstop.

Conventions: all git read operations go through a versioning module over the MNEMO-06 client wrapper
(never the raw SDK), using machine-parseable git output (`--porcelain`, `--pretty=format:`,
`--numstat`, `-z` where it avoids ambiguity) so parsing is robust to filenames with spaces. Restore is
the one *destructive* operation and is therefore conservative: it restores by creating a **new commit**
that returns the tree/file to the chosen revision (so the restore itself is reversible and shows up in
history) — never a hard-reset that discards intervening history. After any restore, the DO graph index
(MNEMO-08) must be re-synced (`reindexAllNotes`) so the index matches the restored tree. Every commit
category from MNEMO-07 (`memory`/`consolidate`/`tool`/`explorer`/`init`) is surfaced so the UI can label
history.
