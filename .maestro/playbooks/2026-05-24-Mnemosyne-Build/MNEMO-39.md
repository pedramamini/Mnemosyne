Phase 39, Track G — Frontend (see `MNEMO-00-ROADMAP.md`). Goal: a versioning view over the agent's brain
git repo — a **commit history list**, a **per-file diff viewer**, and **one-click restore** to a prior
revision. Depends on **MNEMO-12** (brain versioning backend: git history, per-file diff, restore
endpoints), **MNEMO-38** (brain explorer UI + `frontend/src/api/brain.ts` + the Brain tab), and the
scaffold (MNEMO-32). Per `docs/PRD.md` v0.5 §6.9: the brain is a git repo inside the sandbox, auto-committed
on writes and on each consolidation pass; the UI exposes commit history, per-file diffs, and one-click
restore to a prior revision — the safety net consolidation requires and a guard against a bad
self-authored-tool write.

Conventions: extend the existing brain API client and Brain tab from MNEMO-38 rather than creating a new
page. Versioning components live under `frontend/src/components/brain/`. Diffs are rendered with a small
line-based diff renderer (use the design-system primitives from MNEMO-32; only add a diff dependency if one
isn't already present — prefer rendering the unified-diff text the backend returns). Restore is a
destructive-ish action and must require explicit confirmation. Tests use vitest + @testing-library/react
with the API client mocked.
