# MNEMO-27 — Scheduling: per-agent DO timers + Worker cron fan-out + dev trigger route

Phase 27 (see `MNEMO-00-ROADMAP.md`), Track E. Goal: run agents **on a schedule** — per-agent DO
`this.schedule` timers for an agent's own cadence, plus a Worker `scheduled` (cron) handler that **fans out**
to due agents, plus a **dev-only trigger route** because cron does not fire under `wrangler dev`. Depends on
MNEMO-04 (the `MnemosyneAgent` DO + `agent_meta`/schedule state + `idFromName` routing; the DO base provides
`this.schedule`). Per `docs/PRD.md` v0.5 §6.4, §7.4, §8.5: scheduled reporting via DO scheduler + cron
fan-out; **"Cron does not fire in `wrangler dev`" — need a dev-only trigger route.**

Conventions: two layers, kept distinct. (1) The **DO scheduler** (`this.schedule` from the `agents`/`AIChatAgent`
base) is the per-agent timer — an agent schedules its *own* next run and survives hibernation. (2) The **Worker
`scheduled` cron** is the platform heartbeat that wakes the registry and fans out to agents that are due
(the safety net so an idle/evicted DO still fires). Schedule state persists in the DO's `agent_meta` table
(MNEMO-04) and the D1 `agents.schedule_cron` column (MNEMO-02) is the cross-agent listing source the cron
fan-out reads. The scheduled action is "run a headless research turn + delta report" — but this phase only
wires the *plumbing*; the actual run calls the MNEMO-15 headless loop / MNEMO-26 delta report via an injected
callback so scheduling has no hard dependency on those phases landing first.
