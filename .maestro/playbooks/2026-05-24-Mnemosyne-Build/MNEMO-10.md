# MNEMO-10 — Sandbox & Brain: Memory write/append API & consolidation pass

Phase 10 (see `MNEMO-00-ROADMAP.md`). Goal: the write side of memory — a safe write/append API for
notes that keeps the FS, the git history, and the graph index in lockstep, plus the consolidation
pass (the agent's "sleep": re-read, merge, and re-link its own notes), which is **versioned + diffed
before commit** so it cannot silently corrupt the brain. Depends on MNEMO-07 (brain layout + git
`autoCommit` chokepoint) and MNEMO-08 (`GraphIndex.upsertNeuron` / `reindexNote`). Per `docs/PRD.md`
v0.5 §6.2 ("an idle/scheduled consolidation pass lets the agent re-read, merge, and re-link its own
notes — consolidation must be versioned + diffed before commit, it can corrupt the brain otherwise,
backed by the git history of §6.9") and §6.9 (auto-commit on writes and on each consolidation pass).

Conventions: every note write follows the same ordered pipeline — write file (sandbox `writeFile`)
→ reindex the neuron in the DO graph (MNEMO-08) → `autoCommit` with a structured message (MNEMO-07).
That ordering keeps the graph and git in step with the FS. Writes are path-validated through
`src/memory/layout.ts` (no traversal). Consolidation is *proposal-then-apply*: it computes a diff/plan
against the current committed state, surfaces it, and only then applies + commits — so a bad merge is
recoverable via MNEMO-12 restore. Consolidation must run with a clean tree (commit or stash pending
work first) so its commit isolates the consolidation change.
