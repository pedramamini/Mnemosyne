# MNEMO-45 — Messaging: Inbound gateway Worker route

Phase 45 of Track H (see `MNEMO-00-ROADMAP.md`). Goal: the public webhook endpoint Twilio POSTs to when
someone texts an agent's number — it validates the Twilio signature, resolves the destination number to an
`agentId`, normalizes the Twilio form payload into the channel-agnostic `InboundMessage`, hands off to the
per-agent DO, and **acks the webhook immediately** (the actual reply is async, built in MNEMO-46). Depends
on MNEMO-44 (`MessagingChannel`, `TwilioSmsChannel`, `validateTwilioSignature`, `src/messaging/types.ts`)
and MNEMO-02 (`agent_numbers` D1 table + `src/db`). Per `docs/PRD.md` v0.5 §9.3: inbound → gateway Worker:
validate webhook signature, resolve destination number → `agentId`, normalize to a channel-agnostic
message, hand to the existing per-agent DO; per §9.6: validate the Twilio signature on every inbound call.

Conventions: the gateway route is a normal Hono route on the existing app in `src/index.ts`, NOT behind
`requireAuth` (Twilio is the authenticated caller, via the signature — not a logged-in user). It must
**never block on the agent loop** — resolve, normalize, fire-and-forget the handoff, then return Twilio's
expected response immediately. Number→agent resolution reads `agent_numbers` (MNEMO-02). Sender access
control (whitelist / capability tiers) is **NOT** in this phase — it lands in MNEMO-47-a; here we only
authenticate that the call came from Twilio and route it.
