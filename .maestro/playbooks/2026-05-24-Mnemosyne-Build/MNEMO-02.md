# MNEMO-02 — Foundation: D1 schema & migrations

Phase 2 (see `MNEMO-00-ROADMAP.md`). Goal: the relational backbone in Cloudflare D1 — accounts, the agent
registry, report metadata, and messaging tables (schema only, used much later in Track H). Depends on
MNEMO-01 (Worker scaffold + `wrangler.toml` with binding stubs). Per `docs/PRD.md` v0.5 §7.4, D1 holds
accounts, agent registry, and report metadata; per-agent chat/memory/audit state lives in DO SQLite, not here.

Conventions: SQL migrations as numbered files under `migrations/` (`0001_*.sql`, `0002_*.sql`, …). Use
`TEXT` UUID primary keys (app-generated `crypto.randomUUID()`), `TEXT` ISO-8601 timestamps, and explicit
foreign keys. A typed access layer wraps raw D1 with Zod-validated row shapes.
