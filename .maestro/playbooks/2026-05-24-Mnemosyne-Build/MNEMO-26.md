# MNEMO-26 — Reporting: delta-aware report logic (diff against remembered prior state)

Phase 26 (see `MNEMO-00-ROADMAP.md`), Track E. Goal: make scheduled reports surface **what changed** rather
than re-summarizing from scratch — diff the current findings against the agent's *remembered* prior state and
drive the report off the delta. This is the payoff of the memory thesis: because the agent remembers, a weekly
report is "here's what's new/changed since last time," not a cold re-run. Depends on MNEMO-24 (report
generation + the stable-key-order front matter that makes reports diffable) and MNEMO-10 (memory
write/append + consolidation — the remembered prior state). Per `docs/PRD.md` v0.5 §6.4: "Delta-aware:
because the agent remembers prior state, scheduled reports surface *what changed*, not a from-scratch
re-summary."

Conventions: "prior state" has two complementary sources — (1) the prior **report** for this agent/period
(its front matter + structured findings, from the MNEMO-25 D1/R2 archive) and (2) the agent's **memory**
(the neurons the findings were written to, MNEMO-10). The delta is computed over a structured findings
representation, not raw markdown text, so the diff is semantic (added/removed/changed facts) rather than a
line diff. The delta result feeds the existing `generateReport` (MNEMO-24) via the `// MNEMO-26` seam already
left in `src/reports/generate.ts` — this phase fills that seam.

- [x] Create `src/reports/findings.ts`: a structured **findings** model that both the current run and the archive can be reduced to. Export a Zod `Findings` schema — a map/list of typed `Fact`s (`key` (stable id, e.g. `funding.last_round`), `label`, `value`, optional `unit`/`source`/`asOf`), grouped by section. Export `findingsFromReport(frontMatter, markdown | structuredBlock)` that extracts the structured findings persisted with a report (store findings as a JSON block in the report — add it to the MNEMO-24 `ReportInput`/front matter as a `findings` field so it round-trips), and `findingsFromMemory(env, agentId, scope)` that derives current findings from the relevant neurons (via MNEMO-09/10 retrieval). Keep extraction deterministic so two runs over unchanged state produce identical `Findings`.
  - **Done.** `Fact` = `{ key, label, value (always STRING for uniform compare), unit?, source?, asOf?, section? }`; `Findings` = `{ facts: Fact[] }` (flat, key-unique; `groupBySection` reconstructs the by-section view). Findings are persisted as a fenced ` ```mnemosyne-findings ` JSON block — chose a body block over a front-matter `findings` field because the MNEMO-24 YAML serializer is dependency-light (scalars/string-lists only); added `ReportInput.findings` and the markdown assembler embeds the block last (after Related). `findingsFromReport(fm, markdown|Findings)` extracts via `parseFindingsBlock` (malformed/absent ⇒ empty, never throws). `findingsFromMemory(env, agentId, scope, deps?)` reads notes through an injectable `FindingsSource` (default `SandboxFindingsSource` = `find` under NOTES_DIR via shQuote + `readFile`) and merges fact blocks. Determinism: `canonicalizeFindings` sorts + de-dupes by `key`, `serializeFindings` emits a fixed field order.

- [x] Create `src/reports/delta.ts`: the diff engine. Export `diffFindings(prior: Findings, current: Findings): FindingsDelta` returning a typed delta — `added: Fact[]`, `removed: Fact[]`, `changed: { key, from, to }[]`, and `unchanged: Fact[]` (counts only, not echoed). Compare by the stable `key`; for `changed`, compare normalized `value` (trim/format-insensitive where sensible). Export `summarizeDelta(delta): { headline: string; isEmpty: boolean }` producing a one-line human headline ("3 new facts, 1 changed, 0 removed since last report" / "No material changes since last report") for the audit `report.generated` event and the report intro. Keep `diffFindings` pure (no IO) so it is exhaustively unit-testable.
  - **Done.** `diffFindings` is pure: maps both sides by `key`, classifies each into `added`/`removed`/`changed: {key,label,from,to}[]`/`unchangedCount` (the "counts only, not echoed" interpretation — unchanged facts are never carried). `changed` compares `normalizeValue` (trim + lowercase + strip all whitespace) so `"$10M"` ≡ `"$10 M"`. Output lists are ordered by `key` and inputs are never mutated. `summarizeDelta` → `{ headline, isEmpty }`; `isEmpty` is true ONLY when added/removed/changed are all empty (`unchangedCount` doesn't affect it — a brain with 100 steady facts and zero changes is still "empty").

- [x] Create `src/reports/delta-report.ts`: the delta-aware orchestration `generateDeltaReport(env, agentId, scope, opts): Promise<GeneratedReport | null>`. It (a) loads the **prior** report's findings for this agent/period from the MNEMO-25 archive (most recent matching report; if none, treat prior as empty → first run is a full baseline report), (b) computes **current** findings via `findingsFromMemory`, (c) `diffFindings`, (d) builds a `ReportInput` (MNEMO-24) whose sections lead with the delta (a "What changed" section from `added/removed/changed`, charts comparing prior vs current where numeric), embedding `findings` so the *next* run can diff against this one, and (e) calls the existing `generateReport`/`generateAndArchiveReport`. If the delta `isEmpty` and `opts.skipWhenUnchanged` is set (the scheduled-run default), return `null` and emit a `narration`/`milestone` audit event "No material changes — skipped report" instead of generating noise (this protects the §6.4 promise that scheduled reports surface change, not churn).
  - **Done.** `generateDeltaReport(env, agentId, scope, opts, deps?)`: default prior loader reads the most recent archived report's markdown from R2 (`listReportsByAgent` newest-first + `getReportMarkdown` + `findingsFromReport`; none ⇒ empty ⇒ full baseline); default current = `findingsFromMemory`; default generator = `generateAndArchiveReport`. `buildDeltaReportInput` leads with a "What changed" section (New/Changed/Removed bullet lists, only non-empty ones) + a prior-vs-current **bar chart** built from numeric changed facts (`parseNumeric` tolerates `$`/commas/`M`), embeds `findings: current` + `delta`. Empty-delta + `skipWhenUnchanged` ⇒ returns `null` and emits a **milestone-level** `narration` "No material changes — skipped report" (via the generic `emit` passthrough so it surfaces in the calm stream). Every collaborator injectable for container/R2-free testing.

- [x] Fill the `// MNEMO-26` seam in `src/reports/generate.ts`: have the report-generation path accept a precomputed `FindingsDelta` (and `Findings` to persist) so `generateDeltaReport` drives the body, and persist the current `findings` into the report (front matter `findings` field or a fenced JSON block) so it is the diff baseline next time. Update the `report.generated` audit payload to include the delta headline + counts (added/removed/changed) so the glass cockpit shows *why* a report fired (or that one was skipped).
  - **Done.** `ReportInput` gained `findings?` (the markdown assembler embeds the fenced JSON block ⇒ the next diff baseline) + `delta?`. `generate.ts`'s `deltaInfo()` reduces `input.delta` to `{ headline, added, changed, removed }` and both emit sites (`generateReport` + the finalized `generateAndArchiveReport`) pass it to `emitReportGenerated`, which now leads the audit text with the headline and carries `payload.delta`. `audit.ts` extended with `ReportDeltaInfo`. The skipped-report case is recorded by `delta-report.ts`'s milestone narration (above), so the cockpit shows both fired-and-why and skipped.
