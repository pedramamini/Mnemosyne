# MNEMO-15 — Harness & Tools: Agent loop in `MnemosyneAgent`

Phase 15 (see `MNEMO-00-ROADMAP.md`). Goal: the agentic loop itself — `streamText` (interactive) /
`generateText` (headless + scheduled) running inside the per-agent `MnemosyneAgent` Durable Object,
bounded by `stopWhen` with a high deep-research step budget, with message-history persistence and
streaming to the client. Depends on MNEMO-04 (per-agent DO skeleton: `AIChatAgent` base,
`idFromName(agentId)` routing, DO SQLite init) and MNEMO-13 (`getModel()`). Per `docs/PRD.md` v0.5 §7.1
(harness topology A — the DO is the harness host, the model is called via API, the sandbox is a tool
surface; the loop is the Vercel AI SDK loop hosted by `AIChatAgent`) + §7.2 (`getModel()` per-user) +
§8.5 (high step budget for deep research). Mirrors `docs/crema-architecture-reference.md` §2/§7 but
this phase ships with an EMPTY tool map — the tool framework lands in MNEMO-16, terminator in MNEMO-18.

Conventions: the DO class is `MnemosyneAgent extends AIChatAgent<Env>`, declared as a
`new_sqlite_classes` migration and bound `AGENT`, keyed `idFromName(agentId)`. Override `onStart`
(rehydrate), `onConnect` (capture per-connection identity), and `onChatMessage` (the interactive turn)
— no hand-rolled call→parse→execute→feed-back loop. The per-user model is resolved via `getModel(env,
accountId)`; `accountId` is threaded from the owning agent's registry row. The step budget is a named
constant so MNEMO-18's terminator becomes the *deliberate* exit, with `stopWhen` as the hard ceiling.
